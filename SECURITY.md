# Security Policy

## Reporting a Vulnerability

If you believe you've found a security vulnerability in this project, please
report it privately rather than filing a public GitHub issue.

Email **adam@crypto2099.io** and **mad@madorkestra.com** with:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof of concept if available
- Any relevant logs, screenshots, or affected versions

Please do not file vulnerabilities as public issues — this gives us a chance
to assess and patch before details are widely visible.

## What to Expect

We aim to acknowledge reports within **5 business days**. From there we'll
work with you to confirm the issue, develop a fix, and coordinate disclosure
timing.
